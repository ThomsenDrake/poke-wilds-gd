## Warning-tier cases (specific `warning` events)

These are emitted through the existing `warning` event above; they are documented here so the structured payloads are discoverable.

- Save-version refusal — `trace.warning("SaveStore", "Save uses a newer schema; preserved it and starting fresh.", {found_version, supported_version, preserved_path})`. Fired when a save's `version` exceeds `SessionState.SAVE_VERSION`; the unreadable/newer file is renamed to `SAVE_PATH + ".newer.bak"` (reported as `preserved_path`) BEFORE the runtime starts a fresh game and autosaves, so the per-step autosave can never overwrite the preserved newer save. If the preserve rename fails, the refusal is a HARD STOP (`_live_path_protected`): the runtime does not fall through to a fresh-start overwrite of the still-live file.
- Unusable-save preservation — `trace.warning("GameRuntime", "Save parsed but could not be applied; preserved it and starting fresh.", {preserved_path})`. Fired when a version-OK payload fails `apply_loaded_state` (e.g. structurally unusable); the file is renamed to `SAVE_PATH + ".unusable.bak"` before the fresh start, so the parseable-but-unapplicable save is never clobbered either.
- Atomic-rename failure — `trace.warning("SaveStore", "Atomic save rename failed; the previous save is untouched.", {error})`. Fired when the temp-file→final atomic rename in `write_payload` fails; the previous save is left intact and the write returns false.
- Failed-preserve write refusal — `trace.warning("SaveStore", "Save write refused: a newer/corrupt save still occupies the live path and could not be preserved; not clobbering it.", {})`. Fired (once per store) when a newer/corrupt save EXISTS but its preserve rename failed, so the file is still at `SAVE_PATH`; the store refuses every subsequent write rather than rename a fresh save over the un-preserved file. The in-memory game keeps running and persistence resumes on the next launch once a fresh store preserves the file; in this case the `.newer.bak`/`.corrupt.bak` refusal warning above carries `preserved_path=""`.
