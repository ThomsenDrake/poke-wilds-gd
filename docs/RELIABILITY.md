## Automated playtests

```bash
python3 tools/run_playtests.py                    # journey, soak, and all four audits
python3 tools/run_playtests.py --include-smoke    # everything plus smoke scenarios and visual_sweep
```

`playtest_journey` scripts a full player loop (fresh game, overworld steps, a battle played to completion, menu navigation, save round-trip). `playtest_soak` runs a seeded bot (~150 iterations of walking, battles, menu cycles, saves) asserting HP/PP/bag invariants every iteration. `nav_audit` proves traversal agreement (blocked/walkable tiles, field-move gates) and battle/menu navigation reachability with model/cursor consistency. `texture_audit` loads every battle sprite and overworld tile texture through the real pipelines (frame shape, keying). `data_audit` proves encounter-pool battle viability, instance integrity across levels, and bag/catalog resolution. `layout_audit` proves worst-case label fit and cursor alignment across battle and menu screens. Every scenario — smoke or playtest — backs up the player's save file before running and restores it afterward. The runner uses the editor DAP endpoint when available and falls back to headless Godot otherwise; it writes `.godot-smoke/playtest-report.json` and exits nonzero on any failure. Warnings are reported, not failed; exceptions and missing events fail.

Transport honesty: under `PLAYTEST_FORCE_HEADLESS=1`, windowed-only scenarios (`WINDOWED_ONLY_SCENARIOS = {visual_sweep, visual_sweep_update}`, single-sourced in `tools/godot_dap_smoketest.py`) are reported skipped-with-reason, never failed — the summary line reads e.g. `summary: 19/19 (1 skipped-headless)` and the exit code stays 0 whenever only transport skips occurred. `display_matrix` is deliberately NOT in that set: it runs under force-headless and self-skips its pixel work in-engine (`display_matrix.gd:44-47`), still emitting `display_matrix_passed` with `{skipped: headless}` — a distinct, pre-existing in-engine skip, which is why the harness skip count stays 1. (`WINDOWED_SUBPROCESS_SCENARIOS`, which does include `display_matrix`, is only the windowed-subprocess launch set used when NOT force-headless.) `godot_dap_smoketest.py` run directly shares the semantics: it prints a `SKIP:` line, writes an `ok: true` result file with `skipped_reason`, and exits 0.

Report stamps: each run writes `head_sha` (`git rev-parse HEAD`) plus `godot_version`, `window`, and `renderer` harvested from the last `snapshot_captured` trace into `.godot-smoke/playtest-report.json` — absent fields are recorded as `null`, never faked (headless-only runs have all three null, `head_sha` still present). `head_sha` is the freshness-refusal hook for `verify_all.py`. Each run also sweeps stale `.godot-smoke/result-*.json` files older than the run start so stale reds cannot contradict the report. `check_repo_contracts.py` verifies that a present report carries the `head_sha`, `godot_version`, `window`, and `renderer` keys (nulls allowed). On the `visual_sweep` scenario entry the runner additionally records the region-diff verdict fields (`region_failures`, `clusters_explained`, `clusters_unexplained`, `region_artifacts`) and files non-red drift plus unexplained clusters in a report quarantine section as kind `region_drift` — report-tier only, never JSONL (the engine never runs the region diff). From Slice 5 the same entry carries `vision_review_written` (`path`, `shots_reviewed`, `shots_changed`, `findings`, `grounded`, `dropped`, `reviewer` — `shots_reviewed` counts shots COVERED by the review manifest, every baselined shot changed or not; `shots_changed` the subset actually re-reviewed) and kind-`vision_review` quarantine entries (each carrying the Slice-6 `finding_id` join key) when the Lane-4 post-step ran (see Agent vision review below).

## Visual verification

```bash
python3 tools/godot_dap_smoketest.py --project /absolute/path/to/poke-wilds-godot --scene res://scenes/app/Main.tscn --scenario visual_sweep         # compare against baselines
python3 tools/godot_dap_smoketest.py --project /absolute/path/to/poke-wilds-godot --scene res://scenes/app/Main.tscn --scenario visual_sweep_update  # accept new baselines
```

`visual_sweep` captures a deterministic 16-shot set (fixed seed, crafted party including a strip-sprite canary species, fixed wild battle) to `.godot-smoke/shots/` at a canonical 1152x648 window size (applied and restored per run) and diffs it against `docs/generated/visual-baselines/` via `tools/visual_diff.py`; any shot drifting more than 0.5% of pixels fails the scenario. Run `visual_sweep_update` to accept intentional visual changes. Captures require a windowed run (editor DAP launch); under `PLAYTEST_FORCE_HEADLESS=1` windowed-only scenarios are reported skipped-with-reason instead of failing (see Automated playtests above). The diff tool catches pixel drift against what exists — but a human or agent should still READ new shots when states are added, since baselines only guard what already exists.

Explainable per-region diff (Slice 3): every valid sweep shot also writes a canonical semantic sidecar `<shot>.png.sidecar.json` (schema + correlation protocol: [references/snapshot-sidecar.md](references/snapshot-sidecar.md)). After the in-engine global gate (which runs unchanged), `run_playtests.py` runs a region post-step on `visual_sweep` compare runs: `tools/visual_region_diff.py` joins committed baseline sidecars against fresh shot sidecars and fails the scenario on any red-tier region failure — the DECIDUEYE `canary_rect` at ~0 tolerance (first red-tier coded region; catches sub-0.5% sprite changes the global gate is blind to), and near-zero ink/string/label regions (armed only once the noise floor below is measured zero). `tools/visual_explain.py` (imported by the diff) joins each change cluster to the sidecar — `label_moved` / `label_deleted` / `label_overlap_sprite` / `canary_absent` / `canary_shifted` / `palette_dropped` / `region_ink_lost`, or an explicit `unexplained` tag; every cluster carries exactly one. Cursor cells (`cursor_pairs`) are masked as known-dynamic. Artifacts land in `.godot-smoke/region-diff/`: `clusters.json` (machine-readable bboxes + explanations), `<shot>.ascii.txt` (a 32x18 change-density grid an agent reads with no vision), `<shot>.triptych.png` (via the stdlib `tools/png_canvas.py` encoder). Standalone use mirrors the `visual_diff.py` exit contract (0 pass / 1 drift / 2 error):

```bash
python3 tools/visual_region_diff.py --shots-dir .godot-smoke/shots --baseline-dir docs/generated/visual-baselines
```

Region verdicts are recorded by the runner into `.godot-smoke/playtest-report.json` (scenario entry + quarantine section as kind `region_drift`), never the JSONL trace. The post-step is skipped under `PLAYTEST_FORCE_HEADLESS` and in update mode. The DAP transport (`godot_dap_smoketest.py --scenario visual_sweep`) mirrors the post-step in `apply_region_step` (same verdict fields; DAP is currently down, so the mirror runs only when the path is revived).

Region-gate arming (noise floor): the near-zero ink/string tiers arm red only after the duplicate-capture noise floor measures ZERO across 3 consecutive seeded windowed runs — `PLAYTEST_CAPTURE_DUPCHECK=1 python3 tools/run_playtests.py --scenario visual_sweep` three times; each run must report `dup_checked=16`, `max_drift_pct` 0.0, and zero `nondeterministic_pair` records in `user://logs/agent_trace.jsonl` (grep between the run's session boundaries). Until armed, region findings stay quarantine-tier and the nondeterminism is investigated.

All three windowed lanes (`visual_sweep`, `ui_render_audit`'s pixel half, `display_matrix`) use the shared capture contract in `scripts/app/snapshot_capture.gd` (subsystem `vision_fidelity`; spec [product-specs/vision-fidelity.md](product-specs/vision-fidelity.md)): a `RenderingServer.frame_post_draw` readback guard added AFTER each scenario's existing settle waits, plus a validity oracle that classifies a capture as valid or as `blank`/`uniform`/`magenta`/`undersize`/`headless` with a `transport` (headless) vs `regression` (windowed) classification. The lanes consume it at different depths: only `visual_sweep` runs the full `capture()` pipeline, so only its shots emit `snapshot_captured` (with a `trace_cursor` join key into the trace log) on a valid capture; `ui_render_audit`'s pixel half runs the guard + oracle directly and emits quarantine-tier `capture_invalid` only on an invalid verdict (a valid pixel-half shot emits no per-shot trace); `display_matrix` adopts the guard alone around each per-size readback. `capture_invalid` never fails a scenario on its own — except in `visual_sweep`, where an invalid windowed capture still fails red, now with a trace explaining why. A magenta battle SubViewport readback (Godot 4.6 regression #115402, fixed only in 4.7) engages a traced root-viewport-crop fallback in `ui_render_audit`; a magenta `capture_invalid` on the pinned 4.6.1 binary is the guard working, not a sweep defect. Run `visual_sweep` after touching capture/readback paths, window sizing, battle-surface art, or renderer settings; run `ui_render_audit` after touching any UI scene or the battle surface. A duplicate-capture hook (env `PLAYTEST_CAPTURE_DUPCHECK=1` or a scenario option; default off) re-captures each shot and traces any nonzero pair delta as `nondeterministic_pair` with the diff count and first byte offset.

`display_matrix` resizes the window across six sizes (including odd, fractional-inducing ones) and verifies the battle surface renders without scale degradation at each: a block-uniformity check (integer scales produce byte-identical pixel blocks; fractional scales break block periodicity) plus a round-trip content consistency diff. Run it after touching display/layout scaling in any UI.

Baseline regeneration (windowed; run from the repo root, bound each windowed run with a 600000 ms timeout and retry once if a fully-occluded macOS window stalls it): when a sweep's shots legitimately change — intentional art or script changes, or a determinism pin that measurably changes pixels — regenerate all 16 baselines with `python3 tools/run_playtests.py --scenario visual_sweep_update` (windowed-subprocess transport; do NOT set `PLAYTEST_FORCE_HEADLESS` — that turns the scenario into an honest skip, not a regeneration). Stale PNGs are pruned automatically, and sidecars ride the same command: each shot's `<shot>.png.sidecar.json` is co-copied to `docs/generated/visual-baselines/` and stale sidecars are pruned alongside stale PNGs, so PNG↔sidecar never desyncs — regenerate sidecars on exactly the same triggers as baselines (pin/binary/driver change), never by hand. After a regeneration, re-measure the 3-run DUPCHECK noise floor before trusting the region gates again (see Visual verification). Then refresh the tracked import metadata with a headless import pass, `/Applications/Godot.app/Contents/MacOS/Godot --headless --path . --import`: the 16 `docs/generated/visual-baselines/*.png.import` files ARE git-tracked, so review their diff and keep it in the same change (uids persist since source paths are unchanged). Verify the regeneration with TWO consecutive windowed `python3 tools/run_playtests.py --scenario visual_sweep` runs — both must emit `visual_sweep_passed` with max drift 0.0 (bit-identical vs the freshly committed baselines), and their sidecar pairs must be byte-identical after normalizing the TWO documented run-varying fields `ts_msec` (boot clock) and `trace_cursor` (absolute line number into the append-only trace; shifts per run by the prior run's line count — constant across all shots in a run) via `json.load` → pop both → `json.dumps(sort_keys=True)` (policy: [references/snapshot-sidecar.md](references/snapshot-sidecar.md) § Canonical form); optionally `PLAYTEST_CAPTURE_DUPCHECK=1` proves the intra-run noise floor is zero. `tools/determinism_verify.py cmp --a .godot-smoke/shots --b docs/generated/visual-baselines` byte-compares two shot directories without decoding.

Determinism pins make baselines machine-specific: `project.godot` pins `rendering/anti_aliasing/quality/msaa_2d=0` and `rendering/2d/snap/snap_2d_transforms_to_pixel=true` (Slice 2, verify-first — measured evidence and the rejected Nearest/stretch candidate in [product-specs/vision-fidelity.md](product-specs/vision-fidelity.md) § Determinism knobs). Baselines are driver-specific pixels (MoltenVK/Vulkan on this Mac vs the `d3d12` driver pin on Windows), so ANY `project.godot` rendering/display change, Godot binary bump, or driver/renderer change forces the regeneration procedure above. The Slice 1 report stamps (`head_sha`, `godot_version`, `window`, `renderer` in `.godot-smoke/playtest-report.json`, schema-checked by `check_repo_contracts.py`) record baseline provenance: a sweep captured under a different binary or config must not be diffed against these baselines. Mechanical refuse-on-mismatch lands with `tools/verify_all.py` (Workstream L.1); until then the rule is human/agent-enforced policy. `tools/determinism_verify.py pins` asserts the adopted pins are present (and the rejected candidate pins absent); `tools/determinism_verify.py canary --shot docs/generated/visual-baselines/09_battle.png` re-measures the DECIDUEYE canary's block uniformity (pass = 100% 4x4 blocks, 0 off-grid transitions at the canonical window).

### Legibility oracles (Slice 4)

Three coded legibility checks sit on top of the capture contract (full contract + measured evidence: [product-specs/vision-fidelity.md](product-specs/vision-fidelity.md) § Legibility oracles):

- **Glyph template oracle** (`scripts/app/text_oracle.gd`, in-engine): called by `ui_render_audit`'s pixel half per battle state — windowed only (headless `_pixel_half` returns early, so the oracle never runs without a real renderer). It rasterizes each expected string (fonts.ttf@7, black-on-white) in a throwaway SubViewport parked on the audit node, reads the ink mask back after `frame_post_draw`, and XOR-matches it per glyph against the engine readback with budgets calibrated from measured noise: `T_str = 2` px/string and `T_glyph = 1` px/glyph (measured equivalence gap 0, cross-launch 0, white-vs-#f8 fringe ≤1 px/string — 2x headroom over the worst measured fringe). There is no standalone CLI; exercise it via `python3 tools/run_playtests.py --scenario ui_render_audit` (windowed; bind the run with a 600000 ms timeout, never run windowed scenarios concurrently).
- **WCAG contrast** (`tools/contrast_check.py`, stdlib): linearized-sRGB relative-luminance ratio over a label rect (fg = median of sorted distinct ink colors, bg = median of sorted distinct fill colors — conservative by construction), AA threshold 4.5:1 for the 7px battle text. Pure function of pixels: sort distinct colors, take the median of the sorted list — no RNG, no float-ordering nondeterminism, so identical input bytes ⇒ identical ratios. Determinism proof (3 runs, zero variance): run `python3 tools/run_playtests.py --scenario visual_sweep` three consecutive times windowed — Slice 2 proved those captures byte-identical (2/2 bit-identical sweeps, max drift 0.0) — and point the check at each run's shot PNGs + sidecar `labels[].display_rect`; the three ratio vectors must be identical. It is also trivially constant on the committed baselines, which `python3 tools/generate_legibility_report.py --output .godot-smoke/legibility-report.md` exercises in its Contrast section with no Godot. Findings file as quarantine kind `contrast_low` (report-tier via the `visual_sweep` runner post-step; coded RED only after the determinism proof + a clean-run history).
- **CVD simulation** (`tools/cvd_sim.py`, stdlib): Machado, Oliveira & Fernandes 2009 severity-1.0 matrices (IEEE TVCG 15(6), DOI 10.1109/TVCG.2009.113; values from Machado's 2010 thesis, LUME handle 10183/26950; import-time row-sums-to-1.0 assert as a white-preservation + tamper check) simulate the HP-bar triple (#3aa63f / #edba1e / #d03d34) and the sidecar `palettes.canary` under protan/deutan/tritan. Palette-based, so identical on fresh vs committed sidecars — the legibility report's Color-vision deficiency section is the sufficient invocation. Findings file as `cvd_collapse`: QUARANTINE FOREVER — accessibility evidence, never red, never flipped by any clean-run streak.

Recalibration protocol (mandatory after ANY `project.godot` rendering pin, font `.import` parameter, or Godot binary change): re-run the two-launch raster XOR (two separate sequential windowed launches rasterizing the 17 battle strings at 7px, XOR-compared via the repo's stdlib PNG decoder) and re-run the 3-run contrast proof; if the measured noise N rises, raise T from the new N — never hardcode. Refuse to compare calibrations across `godot_version`/`renderer` stamps (the 7px raster is driver/binary-specific: MoltenVK/Vulkan here vs `d3d12` on Windows); the Slice 1 report stamps record which stamp the calibration was measured under.

Graduation procedure (per battle state, Slice 6 rule): record consecutive clean windowed `ui_render_audit` runs with zero `glyph_mismatch` and zero garble/low_ink in the state's rects, plus glyph-oracle agreement; at 5 clean runs flip that state's `GRADUATED_STATES` key to true in `scripts/app/ui_render_audit.gd` and record the evidence — the clean-run entries in the graduation ledger (the recorded-evidence binder documented in the next paragraph), the measurement table and tolerance rule in [product-specs/vision-fidelity.md](product-specs/vision-fidelity.md). Order: `battle_moves` + `battle_item` first (anchor-mode strings — the model fully specifies the integer pen, so the 0-gap proof applies directly), `battle_action` on lint cleanliness (its FIGHT/PKMN/ITEM/RUN strings are baked art, not glyph-matched), `battle_message` last (box-mode — always carries a 1px placement fringe). Menu/party/bag states must NEVER be added to the dict: they use Godot's default theme font at 12/14/20 — a different raster family than fonts.ttf@7, for which the equivalence proof does not hold, and `_audit_menus` has no pixel half anyway. Graduation evidence is valid only at the canonical 1152x648 window (integer battle scale 4); a fractional-scale capture keeps the match quarantine-tier.

Graduation ledger & calibration (Slice 6): clean-run evidence is RECORDED, not asserted — `tools/graduation_ledger.py` (stdlib-only) maintains `docs/generated/graduation-ledger.json`, a TRACKED canonical-JSON (git-diffable) evidence binder whose streaks are COMPUTED from entries. It is a human/agent ritual, deliberately NOT a runner post-step: the ledger is a tracked artifact (a post-step would mutate a committed file on every run — git noise, accidental-commit hazard, unlike the gitignored report/quarantine fields), and `record` needs the machine-local `user://` trace-log join. Historical sessions carry no `head_sha` (single-slot reports, boot-clock `ts_msec`), so there is NO BACKFILL — a streak starts at the first stamped `record`; earlier sessions are citable as prose only. Ritual — sequential, bind each windowed run with a 600000 ms Bash timeout, commit the ledger with the flip:

```bash
python3 tools/run_playtests.py --scenario ui_render_audit && python3 tools/graduation_ledger.py record
python3 tools/graduation_ledger.py status       # read-only: streaks, FLIPPABLE declarations, blockers, next steps
python3 tools/graduation_ledger.py calibration  # once per legibility-garden cycle (weekly, Mon 14:00 UTC); commit the snapshot
```

`record` banks ONE run: it refuses a report without a `ui_render_audit` entry ("re-run first") and a report older than the ledger's last entry (single-slot join hazard), resolves the append-only `agent_trace.jsonl` (macOS `~/Library/Application Support/Godot/app_userdata/PokeWilds-Godot/logs/`; `--trace PATH` overrides for portability — CI has no user:// log), takes the LAST boot-delimited session containing audit events, cross-checks report↔session (both pass events agree with `events_seen`; `quarantined` equals the per-state finding sum; the `text_oracle_passed` payload shows states_checked=4, glyphs_checked>0, T_str=2/T_glyph=1 — any mismatch is an anomaly flag, and an anomaly-flagged entry is INELIGIBLE (untrusted evidence): like other missing-evidence ineligibles it NEITHER COUNTS NOR BREAKS a streak, UNLESS the report is ok=false, in which case it is a windowed RED that breaks every state), computes per-state clean, embeds counts + ≤8 sample payloads (self-sufficient if the JSONL is deleted), and dedupes on the session identity `(start_line, boot_ts_msec)`. A run is ELIGIBLE iff transport ∈ {windowed, dap} AND `ui_render_audit_passed` AND `text_oracle_passed` fired AND the report is ok; a state is CLEAN iff zero findings of kinds `glyph_mismatch`/`garble`/`low_ink`/`clipped`/`forbidden_ink`/`lint_unavailable` (`lint_unavailable` = missing evidence = dirty, never clean). MISSING-EVIDENCE ineligibles (headless CI, or a windowed run with ok=true that lacks a pass event) NEITHER COUNT NOR BREAK streaks, but an ineligible run with EVIDENCE OF A RED (windowed/dap with ok=false — the audit gate failed) BREAKS consecutiveness for every state (a failed windowed audit is not a clean windowed run); streaks reset on an eligible dirty entry and on a windowed red and `status` shows the reset reason (last dirty run id + kinds, with a windowed_red flag) plus stale-evidence detection (streak entries' `head_sha` ≠ current HEAD ⇒ both shas printed, 5 fresh runs needed — progress stays banked).

Flip procedure (when `status` prints FLIPPABLE — `battle_message` is the ONE exception and never prints FLIPPABLE by construction; see below): the trailing 5 eligible entries are clean AND fresh at `git rev-parse HEAD` (the flip is an uncommitted edit atop the evidence HEAD; ANY commit in between stales the streak), predecessor flips exist (`battle_moves`+`battle_item` before `battle_action`; `battle_action` before `battle_message`), then edit ONLY line 16 of `scripts/app/ui_render_audit.gd` (`false`→`true`, in-place constant edit, 0 added lines — the flip changes the verification harness's TIERING, not the game) and write the `flips[]` entry citing `evidence_run_ids` in the same tree. `battle_message` (box mode) is the deliberate exception to "only when FLIPPABLE": its DOCUMENTED box-mode judgment is a FIELD of the flip entry itself, so `status` cannot print FLIPPABLE for it beforehand — it prints DEFERRED (streak progress banked) until the flip records the judgment. It flips LAST, on its OWN fresh 5-clean streak, once the anchor (`battle_moves`+`battle_item`) and lint (`battle_action`) predecessors are in, writing the judgment (engine-owned ~1px pen offset, glyph match excluded by design, lint cleanliness accepted as sufficient, deferral considered) into the `flips[]` entry in the same tree — after which `status` prints GRADUATED, never FLIPPABLE. That edit is app_bootstrap code ⇒ the change contract fires (registry + bootstrap-and-overworld.md + QUALITY_SCORE.md + RELIABILITY.md co-modified). Post-flip proof (perturbations temporary with git-diff revert proof): a seeded defect in the graduated state (the Slice-4 +1px ANCHOR-region shift in `ui_render_model.gd`, which keeps `_check_strings` passing at POS_TOLERANCE 1.5 while breaking the XOR ≫T_str) fails the scenario RED through the graduated gate with no per-bug assertion; the same defect class in a non-graduated state stays quarantine (`_failures` empty); a clean run stays green; `ui_render_model.gd` ends byte-identical to HEAD.

Calibration is free: the existing quarantine→graduation pipeline IS the loop — every pixel finding emits a `quarantine_finding` trace FIRST, and a graduated-state red leaves the same joinable record, so no new engine events. `calibration` (local-only: it needs the machine-local data) appends/replaces a cycle snapshot over runs since the last one: PRECISION = confirmed-true quarantine findings ÷ total in the window (a finding is CONFIRMED iff a later recorded run with the state graduated carries a same-defect-family finding overlapping its stage-px region — `visual_explain.rects_overlap`, the single geometry home; Lane-4 `vision_review` corroboration joins via `finding_id` + shot→state compatibility at the entry's recorded window, counted as corroboration, never as confirmation — Lane 4 never self-confirms), RECALL = graduated-gate catches ÷ real defects (graduated reds ∪ confirmed ∪ recorded seeded-proof plants). Zero denominators are honest: a null rate renders literally as `undefined (N of M — reason)`, NEVER 0.0/1.0/n/a-without-counts. Cycle 1 is the BASELINE; a two-cycle TREND needs the next legibility-garden cycle and is never claimed from one cycle. The garden workflow only RENDERS the committed snapshot — `tools/generate_legibility_report.py`'s "Graduation & calibration" section (absent ledger ⇒ "not yet recorded" note, never an error; calibration rows never count as findings, preserving the first `- Total findings: N` line the Legibility Debt issue regex matches).

Optional vision extra (Slice 6, opt-in): `tools/vision_metrics.py` is an SSIM-map corroboration pass over baseline|fresh shot pairs — advisory localization heat, QUARANTINE-TIER FOREVER, never a gate (full spec: [product-specs/vision-fidelity.md](product-specs/vision-fidelity.md) § Optional vision extra). Install the extra with `uv sync --extra vision` (or `pip install scikit-image` — pyproject `[project.optional-dependencies] vision = ["scikit-image>=0.22"]`); the core tooling stays dependency-free and plain `python3 tools/...` keeps working for everyone else. Standalone: `python3 tools/vision_metrics.py --shots-dir .godot-smoke/shots --baseline-dir docs/generated/visual-baselines --out .godot-smoke/vision-metrics.json` writes schema `vision-metrics/1` (per-16px-window `ssim_divergence` corroboration findings below an advisory 0.85 mean-SSIM floor). It is NOT a runner post-step and nothing it emits flips a result: SSIM windowing averages over 8-16px windows, which dilutes exactly the sprite-scale (1-frame strip offset) and glyph-scale (1-2px XOR) defects the coded oracles catch at exact pixels. Without scikit-image installed the tool prints a skipped note and exits 0, and the legibility report's SSIM section renders a skip note. It is the ONE tool exempted from the core stdlib-only guard (`OPTIONAL_TOOL_EXEMPTIONS` in `tools/check_repo_contracts.py`, which also hardens against any core tool STATICALLY importing it — an AST-level source check that flags a core tool's `import vision_metrics`; a runtime `importlib` load of the file is out of scope for that static guard).

## Agent vision review (Lane 4)

`tools/vision_review.py` auto-runs on EVERY `visual_sweep` compare run — as a `run_playtests.py` post-step (`apply_vision_review`, after `apply_region_gate` and `apply_contrast_cvd`) and standalone — replacing the old manual "a vision-capable reviewer reads every shot" step. It writes `.godot-smoke/vision-review.json` (schema `vision-review/2`; rubric, grounding contract, region-id namespace, and finding schema in [references/vision-review-rubric.md](references/vision-review-rubric.md); full spec in [product-specs/vision-fidelity.md](product-specs/vision-fidelity.md) § Lane 4 automation):

```bash
python3 tools/vision_review.py --shots-dir .godot-smoke/shots --baseline-dir docs/generated/visual-baselines
```

DEFAULT REVIEWER — a deterministic in-process sidecar-consistency checker: the pipeline runs with NO model and is CI-safe (byte-stable findings per seed). It diffs fresh vs committed baseline sidecars (labels deleted/moved/text-changed, cursors missing/moved/appeared, draw-order changes, canary palette drops, canary/expected-region deltas) and reports unexplained change clusters from the region diff — every finding cites a sidecar region id and intersects its bbox BY CONSTRUCTION. It emits nothing for byte-identical shots and nothing groundable for overworld/menu shots lacking sidecar regions (the coverage gap is counted, not faked).

PLUGIN CONTRACT — `--reviewer-cmd "cmd args"` (split via `shlex.split`, never `shell=True`): the command receives per-shot stdin JSON `{shot, shot_kind, paths{before, after, som_before, som_after, crops[], expected_strings, rubric, context}, reviewer_params, finding_schema, region_table, window, clusters, som_legend, grounding_rules}` and must print `{"findings": [...]}` to stdout within the hard timeout (default 300s). `window` (the shot's `[w, h]` frame bounds), `clusters` (its change regions), and `som_legend` (the SoM number→region_id join) are carried in stdin — not only the on-disk `context.json` — so a plugin can construct in-frame, groundable bboxes from stdin alone. Non-zero exit, timeout, or invalid JSON is a TOOL ERROR — exit 2 standalone, red through the runner exception path (fail-closed like every other gate; a hung or garbage plugin is never a silent pass). Per-changed-shot evidence bundles (full-frame before/after byte copies, native-resolution cluster + region crops, numbered Set-of-Mark overlays per arXiv 2310.11441, the expected-strings manifest, the state-group rubric excerpt, context + region table) land in `.godot-smoke/vision-review/<stem>/`.

GROUNDING ENFORCEMENT — after ANY reviewer returns, the pipeline drops + counts every finding that does not cite a resolvable sidecar region id AND intersect one of its rects (`visual_explain.rects_overlap`, the single geometry home): 100% of emitted findings are grounded by construction, and a mock reviewer's ungrounded output is dropped and counted (in the file-level `grounding` stats) with exit still 0 — drops are not errors. Each emitted finding carries a `finding_id` (`vr1-<hex16>` — a stable hash of shot/class/region/values, identical across re-runs, transports, and reviewer implementations) that is recorded on every report quarantine entry of kind `vision_review` as the Slice-6 graduation-ledger join key.

WHEN IT RUNS — every windowed `visual_sweep` compare run, even with 0 changed shots (full manifest, zero findings), so any sweep whose shots change necessarily has a current file. The post-step is skipped under `PLAYTEST_FORCE_HEADLESS` transport and in update/auto_update mode, exactly like the region gate; if a transport-skip run leaves a stale file on disk, the runner REFUSES it — does not record `vision_review_written`, prints a warn (staleness is measured by the shot-hash manifest: sha256 of each covered shot's PNG bytes plus baseline + sidecar hashes — content hashes, mtime-independent). Refuse = ignore + warn + regenerate on the next compare run, NEVER red; only a vision_review tool ERROR (bad PNG decode, reviewer failure, unwritable output) fails red.

TIER — findings are QUARANTINE-FOREVER: recorded in the report quarantine section as kind `vision_review` (report-tier, never JSONL), never failing a run, unless a coded oracle independently confirms the same defect — in which case that oracle's failure governs, and Slice 6's graduation ledger joins the graduated finding to the quarantine `finding_id` it confirms. The gitignored output (`.godot-smoke/vision-review.json` + bundles) is deleted/overwritten unconditionally on first run — the stale July-2026 pilot's bare-array shape fails schema validation and is replaced, not repaired.

## Current risks

- Smoke scenarios depend on a locally running Godot editor exposing DAP on `127.0.0.1:6006`.
- An empty species catalog now skips encounters with a `warning` trace instead of fabricating a synthetic mon; watch for that warning after touching `scripts/data/`.

- Save schema v2 migrates v1 payloads on load; the migration is covered by the `menu_save` and `field_move` scenarios but not by a dedicated fixture test.
