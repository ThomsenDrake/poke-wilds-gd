# Camping, Crafting & Night Survival

Phase 2 turns the Phase-1 placement system ([building-and-placement.md](building-and-placement.md)) into a survivable wilderness loop: the campfire placeholder becomes a LIGHT SOURCE + CRAFTING STATION, the bed placeholder becomes a full heal, a torch becomes a buildable light, crafting gets a faithful campfire recipe table, and night becomes dangerous — without light, Ghost-type Pokemon attack and shadow battles block retreat. The slice is seven new scripts plus one UI scene composed over existing systems: pure-domain `recipes.gd` (recipe table + rules), `material_drops.gd` (interim battle-end ingredient drops), `day_phase.gd` (the single day/night source of truth), and `encounter_selection.gd` (level/fallback helpers extracted out of `game_runtime.gd`); runtimes `night_system.gd` (light model, ghost pool, shadow-retreat gate), `camping_runtime.gd` (rest/heal/campsite + the campsite accessors extracted out of `game_runtime.gd`), and `crafting_runtime.gd` (consume/grant at a station); the night glow is composed INSIDE `structure_layer.gd` (the design's documented fallback path — the camping visuals never depend on which compositor landed); and `scripts/ui/camp_menu.gd` + `scenes/ui/CampMenu.tscn` (the campfire menu). Every budget-full file is touched only by extraction or same-line edit: `game_runtime.gd` (320/320) KEEPS its budget by extracting the campsite accessors and the encounter level/fallback helpers FIRST (one-line forwarders, static call sites) and spending the freed lines on the new wiring; `main.gd` (220/220) gets exactly ONE in-place string edit; `pokemon_rules.gd` (320/320) stays untouched (the evolution fix is caller-side); `battle_runtime.gd` (319/320) moves the one-line `_move_priority` into `battle_rules.gd` (static `BattleRules.move_priority`) to pay for the time-context and retreat-callable additions, staying within the 320 budget.

## Supported behavior

- The campfire is a crafting station AND a light source; the bed is a full heal; the sleeping bag is a weaker, status-blind heal used from the bag. Resting establishes the campsite anchor that the overflow-mon hold and blackout return both read.
- Crafting consumes bag materials at a faced, lit-or-unlit campfire through a campfire menu listing exactly the five campfire recipes; refusals are traced with their missing counts, never silent.
- Night (minutes >= 1230 or < 270, mirroring the world view's tint keyframes) without an active light source within Manhattan distance 4 — a lit campfire, a torch, or a Flash-capable (Fire-type) party member — spawns Ghost-type encounters at half the step-encounter cadence, and those shadow battles block retreat for their duration.
- Encounter selection gains a time-of-day filter (Umbreon is a night-only Savanna spawn; every biome's night pool admits Ghost types), and the battle evolution check receives real time context so the Espeon/Umbreon/Frosmoth gates work.
- All constants below are PINNED (see § Pinned constants) and the scenarios assert their boundary values against the presentation keyframes so the logic and the tint cannot drift silently.

### Campfire station and lit state

- The campfire keeps its Phase-1 definition (`structures.gd`: cost 4 log + 2 dry_soil, non-walkable, "A campfire blocks the way.", a 2-frame sheet of which one 16x20 frame is the static prop). Phase 2 pins BOTH frames: the static prop frame (`_CAMPFIRE_REGION`) and `_CAMPFIRE_LIT_REGION := Rect2(16, 0, 16, 20)` (the lit frame of `pokewilds/tiles/campfire1.png`). `sprite_region_for` gains a `lit` parameter defaulting true (`_CAMPFIRE_LIT_REGION if lit else _CAMPFIRE_REGION`); `world_overrides.apply_placement` threads the placement entry's `lit` field through `Structures.placement_is_lit` into that call, so an extinguished fire stamps the UNLIT base frame as well as dropping its glow (sprite, glow layer and the night system's light model all read the same placement field — they can never disagree).
- A placed campfire is LIT BY DEFAULT: the original has no fuel mechanic (the wiki Campfire page describes the recipe list only; firelight is permanent — "light or die" is about PLACEMENT, not fuel). Lit is a runtime toggle carried as an extra `"lit": false` field on the placement entry (ABSENT = lit, so every Phase-1 campfire save loads as lit for free). `LIGHT_SOURCES := {"campfire": true, "torch": true}` + the `is_light_source(id)` helper are the shared light-source set the light model reads.
- An EXTINGUISHED campfire is still a crafting station but emits no light (extinguish/re-light is a camp-menu toggle; the toggle mutates the generator's live placement entry, traces `campfire_lit`, and saves via the existing save path). No separate session-level light state exists — the placement map is canonical, exactly like occupancy.

### Torch

- The torch is a buildable light structure APPENDED LAST to `structures.IDS` (after `bed`), so every existing build-cycle index and scenario that calls `try_place` by explicit id stays stable. Cost `{log: 1, dry_soil: 1}` — faithful to the original "1x Log & 1x Grass" (`.firecrawl/buildings-scrape.md:286`) under the documented Phase-1 Grass→`dry_soil` mapping ([building-and-placement.md](building-and-placement.md) § Structures and costs). Asset `res://pokewilds/tiles/torch_sheet1.png`.
- The torch is non-walkable like the campfire (block reason "A torch blocks the way."), ground-placeable via the existing `can_place_on`, and ALWAYS LIT (no extinguish — a static light source, so it carries no `lit` field). The original also allows torches on fences and interior walls (`buildings-scrape.md:288`); the port's one-structure-per-open-tile placement model cannot express mounting, so fence/wall mounting is a DOCUMENTED DEVIATION (Phase 3 building interiors).

### Crafting at campfires

- Recipe data + rules live in pure domain (`scripts/domain/recipes.gd`, stdlib-only): `STATION_CAMPFIRE := "campfire"`, `STATION_KILN := "kiln"`, a `RECIPES` dict keyed by lowercase bag ids (every id already resolves in `pokewilds/i18n/item.properties`), and four pure functions — `recipe_for(output_id)`, `craftable_at_station(station_id)` (sorted Array), `missing_ingredients(recipe, bag_counts)` (`{item_id: need}`), `can_craft(output_id, bag_counts, station_id)`.
- The campfire menu lists EXACTLY `craftable_at_station("campfire")` — five recipes — with missing counts greyed. There is NO recipe discovery/learned state (the wiki has no learning mechanic), so all five are always listable and nothing about recipes persists.
- Crafting is all-or-nothing: `crafting_runtime.craft(output_id, station_id)` validates the recipe exists, the station matches, and the bag covers every ingredient (checked BEFORE the first removal), then consumes via `session.remove_item` and grants via `session.add_item`, emitting `item_crafted {output_id, station, ingredients}` on success or `craft_refused {output_id, station, reason, missing}` on failure (`reason` ∈ `no_recipe`, `no_station`, `wrong_station`, `missing`). Crafting Great Ball at a campfire is refused `wrong_station` ("That needs a kiln.") — see the kiln gate in § Phasing decisions. `crafting_runtime.missing_for(output_id)` feeds the menu's greyed short-counts.
- Craft-into-bag only: a crafted Poke Ball enters the bag, but the battle's throw ball is hardcoded (`battle_runtime` BALL_ID = `poke_ball`) until the Phase-8 ball tiers — crafted ball tiers are bag inventory in Phase 2 (documented in § Scope boundary).

### Ingredient source (interim battle-end type drops)

- The recipes above need magnet, hard_shell, metal_coat, silky_thread, and soft_feather. The ORIGINAL sources them from happy-Pokemon habitat drops (a Phase 5 system the port has not built yet). Phase 2 ships a minimal deterministic INTERIM so the crafting loop is playable end to end: `scripts/domain/material_drops.gd` maps a defeated/captured species' primary type to one material, granted by `game_runtime` on battle victory/capture with a `material_dropped {species_id, item_id}` trace.
- `TYPE_MATERIALS := {"ELECTRIC": "magnet", "WATER": "hard_shell", "STEEL": "metal_coat", "BUG": "silky_thread", "FLYING": "soft_feather"}` — the five materials Phase-2 recipes consume (charcoal EXCLUDED: no Phase-2 recipe uses it), mapped 1:1 from the wiki's species/type drop table (`.firecrawl/wiki-materials.md`: Electric lines drop Magnet :484-486, Water lines Hard Shell :466-468, Bug lines Silky Thread :469-471, Flying secondaries Soft Feather :475-477/:465, Steel lines Metal Coat :543/:571). `drop_for(species_entry) -> String` walks the species' types in PRIMARY→SECONDARY order and returns the first mapped material (so dual-type mons match the wiki rows — Pidgey's soft_feather comes from its secondary FLYING type), else `""`; `is_drop_material(item_id)` is the audit seam for the witness assertion below.
- The drop is PURE and RNG-FREE: 100% on a type-matched victory/capture, no roll — so it needs NO `seed_for_smoke` seam and cannot shift the shared rng stream. LOAD-BEARING INVARIANT: `TYPE_MATERIALS` NEVER contains `log` or `hard_stone` — those are the demolition-witness materials the region-seal escape in [building-and-placement.md](building-and-placement.md) § The would-trap guard rides on (log→Cut, hard_stone→Smash). `craft_flow` asserts the invariant (via `crafting_runtime.drop_witness_clean` + `is_drop_material`) on every run, so a slip fails loudly; Phase 5's faithful habitat drops retire this interim (tech-debt entry + § Phasing decisions).

### Rest and the heal model

TWO rest paths, faithful split — there is NO "sleep at the campfire" action (campfire = light + crafting; sleep = bag or bed).

- SLEEPING BAG — used FROM THE BAG (faithful: "This item is in your bag at the start of the game. Your Pokemon regain health while you sleep in it." — `.firecrawl/fresh-sleepingbag.md:145`). It rides `STARTING_BAG` as `sleeping_bag: 1` (`session_state.gd:24-27`, +1 line; the id already resolves in i18n, so `data_audit`'s starting-bag resolution passes). It is a REUSABLE key item — never consumed on use. `bag_screen.gd` `_activate_item` gains one branch (`item_id == "sleeping_bag"` → the rest action, message from the result). Effect: HP-ONLY — every party member heals `ceil(max_hp * HEAL_FRACTION_BAG)` with `HEAL_FRACTION_BAG := 0.5`, fainted members revive to 50% HP, and status is NOT cured. The faithful answer to the spec's "or partial" open question is NO CURE: every scraped source puts status removal on beds/berries/faint-then-bag (the bag revives but does not cure live status — `.firecrawl/fresh-sleepingbag.md` status list: Persim cures Confusion :278, Pecha cures Poison :290; beds are the resting cure).
- BED — interact (`Z` facing a placed bed) routes through `field_action_router` directly to `rest("bed")` (the bed has no craft menu). Effect: FULL party heal + status cure + revive fainted — `heal_party_full()` semantics (`session_state.gd:179`: HP=max, status="", sleep_turns=0 for ALL members including fainted). Heal logic lives in `camping_runtime.gd` mutating the session party dicts directly (the `bag_screen._apply_potion` pattern); the session API itself is unchanged (the only session_state additions are the `sleeping_bag` starting-bag entry and the one-line `time_of_day_label()` delegation the evolution fix reads).
- The original's bag RESTRICTIONS — "cannot be used when on certain terrain or when you are near an aggressive Pokemon" (`fresh-sleepingbag.md:145`) — are DEFERRED to Phase 6 (no aggressive-chase system is ported), and the indoor-effectiveness bonus is DEFERRED with the enclosure flood-fill (Phase-1 deferral; the bag is flat 50% — documented deviations).

Both ride the existing `session.advance_time(delta)` (`session_state.gd:233`).
- CAMPSITE ESTABLISHMENT — the missing write: `rest()` sets `session.campsite_tile = session.player_tile` and traces `campsite_established {tile, kind}`. Today only `reset_for_new_game` writes `campsite_tile` (`session_state.gd:53`, defaulting to spawn); the overflow-mon relocation already READS it, so resting makes the 0.1 anchor REAL (overflow mons relocate to where you last slept). Rest also traces `rested {kind, tile, minutes_advanced, woke_at, healed, revived}`.
- BLACKOUT RETURN: `battle_runtime.gd:294` `_session.player_tile = Vector2i.ZERO` becomes `_session.player_tile = _session.campsite_tile` (same line length, zero budget impact). ### Night danger: light or ghosts

- PHASE DETERMINATION — `scripts/domain/day_phase.gd` is the single source of truth, mirroring `world_view.gd` `TIME_OF_DAY_KEYFRAMES` (:10-19): `NIGHT_START := 1230`, `NIGHT_END := 270`, `DAWN_END := 480`, `DUSK_START := 1020`. `is_night(minutes)` → `minutes >= 1230 or minutes < 270` (night ≈ 20:30–04:30). `time_of_day_label(minutes)` → `"NIGHT" | "DAY"` (dawn/dusk/day ALL map to `"DAY"` — documented; Espeon's `TR_MORNDAY` is morning+day hours). `phase_for(minutes)` → `"night"|"dawn"|"day"|"dusk"` for tint-adjacent queries. The world view keeps its keyframes (presentation untouched) with a comment pointing at `day_phase`; `night_cycle` asserts the boundary minutes (269/270/1229/1230) AGREE with the tint keyframes so the duplication cannot drift silently.
- LIGHT MODEL (`night_system.gd`): `LIGHT_RADIUS := 4` (Manhattan; undocumented in the originals — "Flash shares the same range as a campfire", so ONE constant covers all three sources; pinned as a design note). `has_light_at(tile)` is true iff (a) a placed campfire with `lit != false` OR a torch exists within Manhattan 4 (read via an injected Callable into the generator's placements map — `_world_gen.placements_for_save`; `game_runtime.placed_structures()` exposes the same map), or (b) any party member satisfies `FieldMoves.can_perform(mon, "flash", get_species)` — `AUTO_TYPES["flash"] == "FIRE"` already exists (`field_moves.gd:6`), so a Fire-type in the party is PASSIVE light per the dev interview ("If you have a fire type that's giving off light, that's going to illuminate the area around you. Um if you're not illuminating it, you might be attacked by ghosts at night." — `.firecrawl/youtube-interview.md:246-250`). This is a capability READ, not the Phase-4 Flash field-move ACTION — the split is documented in § Scope boundary.
- GHOST ENCOUNTERS: `game_runtime._pick_encounter_species` (`game_runtime.gd:223`) delegates to `night_system.try_ghost_species(session.player_tile)`: only when `is_night()` AND `has_light_at` fails AND `_rng.randf() < GHOST_CHANCE` (`:= 0.5`, pinned) does the night system pick a ghost, trace `night_hazard_spawned {species_id, tile, minutes}` (source `NightSystem`), and mark a PENDING shadow battle — otherwise it returns `""` (day and lit draws consume NO rng, so the daytime encounter stream is byte-identical to pre-Phase-2 behavior). The next `start_wild_battle` resolves the mark via `night_system.begin_battle(wild_mon)` — a non-ghost encounter CLEARS it, so the retreat block can never leak into a later, ordinary battle. The roll rides the EXISTING shared `_rng` stream (the player avatar's `encounter_chance = 0.12` per step, `player_avatar.gd:21`, decides WHETHER an encounter fires; the ghost roll decides whether a night encounter is a ghost — 50% of the cadence), so `seed_for_smoke` (`game_runtime.gd:202-204`) pins everything with no new seam. GHOST POOL: type-derived — `BIOME_TYPES` has no GHOST entry, so `night_system` lazily builds a SORTED list of battle-viable Ghost-type species from the catalog (via the injected `biome_encounters.is_battle_viable` callable, so the two pools never disagree about what is encounterable; ~50 species incl. GASTLY/DUSKULL/DREEPY lines), and selection is sorted-ids + `randi_range` on the shared rng → deterministic under seed. DOCUMENTED SIMPLIFICATION: the original gates ghosts on Damp/Mystic/Cursed Soil DWELL TIME ("They can only trigger when the player is on Damp, Mystic, or Cursed Soil for a certain amount of time. Sources of light will stop them… Campfires, torches, lava, Pokemon with Flash, or just waiting until daytime." — `.firecrawl/wiki-overworld-encounters.md:274`) — tiles the port does not generate — so the reviewer/dev-attested framing wins: ANY night without active light.
- SHADOW RETREAT BLOCK: `battle_runtime.run_from_battle` (`battle_runtime.gd:128-132`) gains one injected check — a `retreat_check` Callable (setup parameter, default empty = always allowed; `game_runtime` passes its `_retreat_allowed` forwarder → `night_system.retreat_allowed()`). While `battle_is_shadow()`, retreat is blocked for the DURATION of the shadow battle and the night system emits `retreat_blocked {species_id}` (source `NightSystem`) once per battle. Dawn-cannot-arrive-mid-battle is a documented port reality (time advances 1 min/step and no steps occur in battle), so "until dawn" ports as "for the whole shadow battle": the escapes are victory, capture, or blackout (→ campsite). Between battles, dawn or light clears the hazard naturally (no new ghost starts in day or in light). `battle_runtime` never imports `night_system` — the injected callable keeps the layering.

### Light rendering

- The glow is composed INSIDE `structure_layer.gd` — the design's DOCUMENTED FALLBACK path, chosen so the camping visuals never depend on which compositor landed (the alternative was a separate `light_layer.gd` child of `world_view.gd`; neither `world_view.gd` nor `main.gd` is touched): one static warm radial-gradient sprite (`GLOW_SIZE := 48`, `GLOW_COLOR := Color(1.0, 0.82, 0.5, 0.5)`) per lit campfire/torch tile in the visible window — a GENERATED `ImageTexture`, additive blend, NO per-frame animation (the byte-stability contract the sweep depends on; animated flicker would flake the baselines) — night-only (global visibility modulated by `DayPhase.is_night`, refreshed when the clock crosses the night boundary), refreshed on `world_overridden` (the campfire lit toggle mirrors that signal) and on player tile change. The light layer reads lit placements via the runtime's placement map exactly like the night system's light model, so render and gameplay can never disagree about which fires glow.

### Camp menu routing (Z-precedence, witness-safe)

- Today `Z` on a faced placed campfire DEMOLISHES (`game_runtime.harvest_tile` → `try_demolish`). ## Phasing decisions (faithful recipes and documented deviations)

1. INGREDIENT SOURCE (the critical path): ship the minimal interim battle-end type drops (above).
